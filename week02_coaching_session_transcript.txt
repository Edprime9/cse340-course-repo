Please paste the Node.js function from your projects.js file that retrieves the project data from the database, and explain the flow of how the data moves from the database into your application.

User:
import db from "./db.js";

async function getAllProjects() {
  const query = `
        SELECT 
            service_projects.project_id,
            service_projects.title,
            service_projects.description,
            service_projects.location,
            service_projects.project_date,
            organization.name
        FROM service_projects
        JOIN organization
        ON service_projects.organization_id = organization.organization_id
        ORDER BY service_projects.project_date;
    `;

  const result = await db.query(query);
  return result.rows;
}

export { getAllProjects };

Assistant: Well done, Morrison Eli. This is a solid example of separating database logic into its own module, which keeps your application organized and easier to maintain.

Your data flow works like this:
1. The getAllProjects() function is called from another file, such as server.js
2. The SQL query is stored inside the query variable
3. db.query(query) sends the SQL command to the database
4. await pauses execution until the database responds
5. The database returns the matching rows
6. result.rows extracts the actual data
7. The function returns that data so the route handler can use it

You also used ORDER BY service_projects.project_date, which is a good design choice because it keeps the projects sorted by date before they reach the frontend.
